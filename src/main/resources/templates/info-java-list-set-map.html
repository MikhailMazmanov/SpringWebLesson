List, Set, Map — в чём разница?

Эти три типа — основа Java Collections Framework. У каждого своё предназначение:

  • List — упорядоченная коллекция с доступом по индексу.
  • Set — набор уникальных элементов.
  • Map — хранение данных в виде пар ключ-значение.

List — список с порядком

Элементы в List сохраняют свой порядок добавления и могут дублироваться. Каждый элемент имеет индекс.


import java.util.List;
import java.util.ArrayList;

List inventory = new ArrayList<>();
inventory.add("меч");
inventory.add("щит");
inventory.add("щит");

System.out.println(inventory);

Результат:

[меч, щит, щит]

Set — только уникальное

Set не допускает повторяющихся элементов. Порядок может быть случайным или заданным (например, через TreeSet).


import java.util.Set;
import java.util.HashSet;

Set uniqueItems = new HashSet<>();
uniqueItems.add("меч");
uniqueItems.add("щит");
uniqueItems.add("щит");

System.out.println(uniqueItems);

Результат:

[меч, щит]

Map — связь ключ-значение

Map используется для хранения ассоциаций. Ключи должны быть уникальными.


import java.util.Map;
import java.util.HashMap;

Map, Integer> prices = new HashMap<>();
prices.put("меч", 100);
prices.put("щит", 80);
prices.put("меч", 90); // Перезаписывает старое значение

System.out.println(prices.get("щит"));

Результат:

80

Когда что использовать?

Тип Порядок Дубликаты Индекс Пример использования
List Да Да Да Список задач
Set Нет Нет Нет Уникальные теги
Map Нет Ключи нет, значения да Нет Словарь

Stream API и коллекции

Java 24 позволяет удобно обрабатывать данные из List, Set и Map с помощью Stream API:


import java.util.stream.Collectors;

String result = inventory.stream()
    .filter(item -> item.startsWith("м"))
    .collect(Collectors.joining(", "));

System.out.println(result);

Результат:

меч

Знание различий между List, Set и Map помогает правильно выбирать структуры данных для решения конкретных задач.
